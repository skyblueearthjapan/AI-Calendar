# AIカレンダー 設計仕様書
（AI家計簿シリーズ）

## 1. 概要

本アプリケーションは「AI家計簿」と同一シリーズとして開発する
**音声入力対応のAIカレンダー & 日付別メモアプリ**である。

- UIトーン：可愛らしい / 淡色 / 丸みのあるデザイン
- 操作の中心：音声入力（OpenAI API利用）
- ページ構成：**カレンダー / メモ の2ページのみ**
- モバイル（特にiPhone）最優先設計

---

## 2. ページ構成

### 2.1 カレンダーページ（メイン）

#### 目的
- 月間カレンダーを大きく表示
- 音声入力で予定を追加・更新する

#### レイアウト
- 上部：音声入力UI
  - 大きな丸いマイクボタン（AI家計簿と同デザイン）
  - カレンダー表示を邪魔しない高さ
- 中央〜下部：月間カレンダー
  - 日本式（日曜始まり）
  - 1日1マスを大きく（モバイルで視認可能）

#### 表示内容（1日）
- 日付
- 六曜（可能であれば）
  - 大安 / 友引 / 先勝 / 先負 / 赤口 / 仏滅
- 予定タイトル（複数ある場合は省略表示）

---

### 2.2 メモページ

#### 目的
- 日付ごとのメモをシンプルに残す
- 音声入力＋AI整形で「読みやすい自然文」にする

#### レイアウト
- 上部：日付ナビ
  - ◀ 前日 / ▶ 翌日
  - 中央に日付表示（YYYY/MM/DD）
  - 日付タップで任意日へジャンプ
- 中央：メモ入力欄（大きなテキストエリア）
- 下部：音声入力マイクUI
- メモ欄右上：**コピー（全文コピー）ボタン**

---

## 3. 音声入力仕様（共通）

### 3.1 フロー
1. マイクボタン押下 → 録音開始
2. 再押下 → 録音停止
3. 音声を文字起こし
4. AIで解析・整形
5. ユーザー確認
6. 確定後、保存

---

## 4. カレンダー：AI解析仕様

### 4.1 入力
- ユーザーが話した自然言語テキスト

### 4.2 AIが抽出する項目
- タイトル（必須）
- 日付（必須）
- 時間（任意）
  - 開始のみ / 開始-終了 / なし
- メモ（任意）
- 期間指定
  - 「◯月◯日まで」「◯日から◯日まで」等

### 4.3 データ構造（例）

```json
{
  "title": "歯医者",
  "date": "2026-01-15",
  "start_time": "15:00",
  "end_time": "16:00",
  "memo": "保険証を持参",
  "all_day": false
}
```

期間予定の場合：

```json
{
  "title": "出張",
  "range": {
    "start_date": "2026-01-20",
    "end_date": "2026-01-22"
  },
  "memo": "新幹線は後で確定"
}
```

### 4.4 AI判断ルール
- 時間が曖昧 → 時間なしで登録
- タイトルが不明 → ユーザーに確認
- 期間指定 → 複数日へ反映

---

## 5. メモ：AI整形仕様（重要）

### 5.1 目的
- 音声メモに含まれる雑音を除去し、自然な文章に整える
- 要約・フォーマット化は行わない

### 5.2 やること
- フィラー削除
  - 例：「えー」「あのー」「そのー」
- 言い直しの整理
- 句読点・改行を最小限補正

### 5.3 やらないこと
- 要約しない
- 箇条書きにしない
- 意味を変えない
- 情報を追加しない

### 5.4 メモ反映ルール
- デフォルトは 追記
- 既存メモ末尾に `\n\n` を挟んで追加

---

## 6. メモ機能仕様

### 6.1 データ構造
- key：日付（YYYY-MM-DD）
- value：メモ本文（string）

### 6.2 履歴の考え方
- 履歴は「日付移動」で表現
- バージョン管理は行わない

---

## 7. コピー機能仕様
- メモ欄右上に「コピー」ボタン
- 押下で全文コピー
- 文字選択不要
- コピー成功時にトースト表示

---

## 8. UI/UX 方針
- 淡いパステルカラー
- 角丸・柔らかい影
- 大きなタップ領域
- マイクUIはシリーズ共通アイコン

---

## 9. MVP範囲（初期実装）

### 必須
- 月間カレンダー表示
- 音声入力で予定追加
- 期間予定対応
- メモ（日付別）
- メモ音声入力＋AI整形
- メモ全文コピー

### 後回し可
- 六曜表示（実装難易度次第）
- iPhoneメモ連携
- 高度な予定編集UI

---

## 10. 非機能要件
- モバイル最適化必須
- 音声入力前にマイク権限確認
- 誤登録防止の確認ステップを必須とする

---

## 11. 補足

本設計仕様書は
「コーディングエージェントが迷わず実装できること」
を最優先に作成されている。

UI判断・AI挙動は、本仕様を正とする。
